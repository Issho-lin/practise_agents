 1. 多模态客服 Agent（感知 + 理解 + 交互）

  需求：用户发一张产品照片 + 一段语音/文字描述问题，Agent 能识别产品、理解问题、检索知识库、给出解决方案，并生成图文回复。

  覆盖的知识点：
  - 多模态输入处理：图像分类/目标检测（CLIP、Detr）、语音转文字（Whisper）、文本 NER
  - 意图识别 + 实体抽取：从混合输入中提取"我要退这台咖啡机，型号 X200"
  - 工具路由（Router）：判断是售后/咨询/投诉，路由到对应子流程
  - 知识库检索：产品信息、退换货政策、常见问题
  - 多模态输出：文字回复 + 操作步骤截图标注 + 可选语音回复
  - 上下文管理：多轮对话中保持产品、问题状态的上下文
  - 情感分析：识别用户情绪，愤怒时升级人工客服
  - 记忆系统：记住用户历史工单、购买记录、偏好

  2. 智能投研 Agent（搜索 + 推理 + 结构化输出）

  需求：用户说"帮我分析比亚迪最新季度的表现"，Agent 自动抓取财报、新闻、竞品数据、行业报告，生成一份结构化投资分析报告，含风险提示。

  覆盖的知识点：
  - 多源信息聚合：财报 API（yfinance/SEC EDGAR）、新闻 API（Tavily/Bloomberg）、社交媒体情绪
  - 文档解析：PDF 财报解析、表格提取、图文混合文档处理
  - 信息交叉验证：不同来源的信息冲突时如何仲裁
  - 推理链构建：从多个信号推导结论（营收增长 + 毛利下降 + 竞争对手动作 = ？）
  - 结构化报告生成：不只是文字，还要有指标卡片、趋势表格、对比矩阵
  - 不确定性量化：明确标注"数据截止 X 月"、"预测基于假设 Y"
  - 反事实推理："如果碳酸锂价格下跌 20%，对利润的影响？"
  - 可追溯性：每个结论都要能溯源到具体数据来源

  3. 教育辅导 Agent（对话 + 推理 + 自适应）

  需求：学生问一道数学/物理题，Agent 不能直接给答案，而是通过苏格拉底式引导，一步步问问题帮助学生自己想出来。

  覆盖的知识点：
  - 问题分解：理解题目考点（"这道题考的是动量守恒和能量守恒"）
  - Socratic 对话策略：不直接给答案，用追问引导（"你觉得这里应该用哪个公式？为什么？"）
  - 学生模型（Student Modeling）：根据学生回答推断知识盲区
  - 自适应难度：学生答对了升难度，答错了降难度给铺垫题
  - 解题路径推理：一题多解，根据学生已学知识选择合适路径
  - 错误诊断：学生答错了，判断是公式用错还是计算错误还是概念混淆
  - 反馈生成：具体指出哪里错了、正确思路是什么、类似题巩固
  - 长期学习轨迹：记住哪些章节薄弱，后续主动出对应练习

  4. 会议纪要与行动追踪 Agent（感知 + 结构化 + 任务编排）

  需求：接入会议录音/直播转录，实时识别发言人和话题切换，生成结构化纪要，自动提取 Action Item 并分配到对应人的任务系统。

  覆盖的知识点：
  - 语音流处理：实时 ASR（WebRTC VAD 说话人检测）、流式转录
  - 说话人分离（Diarization）："谁在什么时候说了什么"
  - 话题分割：自动切分会义议程（"下面讨论 Q3 预算" → 新 segment）
  - 信息结构化提取：从口语化表达中提取结构化信息（"那下周三之前把方案给我" → {action: 提交方案, owner: 对方, due: 下周三}）
  - 决策/争议/待办 三元分类：一句话是决策？争议？还是 Action？
  - 冲突消解：两个人对 Action 的理解不一致时，在会中澄清
  - 多轮跟进：Action 到期前自动 reminder，完成后自动更新状态
  - 语义摘要 vs 逐字稿：同一份会议同时生成高层摘要和详细记录

  5. 个人生活管家 Agent（长期记忆 + 规划 + 多工具）

  需求：一个懂你的私人助理——"我下周要去东京玩 5 天，预算 2 万，帮我规划行程 + 订酒店 + 记到日程里"。

  覆盖的知识点：
  - 用户画像建模：长期记住偏好（不吃辣、怕高、喜欢历史还是购物）
  - 复杂任务规划（Planning）：行程规划 = 约束满足问题（时间 + 预算 + 距离 + 兴趣）
  - 多工具调用：日历 API、地图 API、酒店搜索、航班搜索
  - 约束优化：景点之间的交通时间、预算分配、开放时间
  - 多轮协商：用户说"太贵了"→ 自动重新优化预算分配
  - 实时状态感知：天气变化、航班延误 → 动态调整计划
  - 主动提醒：出发前一天提醒带护照、航班值机提醒
  - 隐私与权限：敏感信息（护照号、信用卡）的加密存储，第三方 API 的权限范围

  6. 法律/合规分析 Agent（专业领域 + 推理 + 可解释性）

  需求：用户上传一份合同或规章制度，Agent 识别风险条款、对比法律法规、给出合规建议和修改方案。

  覆盖的知识点：
  - 长上下文处理：合同可能几十页，需要 efficient 的 context management（RAG + sliding window）
  - 专业术语理解：法律领域特有的术语和表述（"不可抗力"、"连带责任"、"竞业限制"）
  - 法规检索：自动关联相关法条（《民法典》《劳动合同法》）
  - 风险分级：条款风险 = 高/中/低，附具体依据
  - 条款对比：本合同 vs 标准模板 vs 法规要求，差异在哪
  - 修改建议生成：不只是指出问题，还给出改写后的条款
  - 可解释性：每个风险判断都要引用法条原文 + 解释逻辑
  - 版本追踪：合同多轮修改后，自动标记新增/删除/修改的条款

  7. 创意写作 Agent（生成 + 风格控制 + 迭代）

  需求：用户输入一个主题和风格要求（"写一个赛博朋克风格的悬疑短篇小说，类似《银翼杀手》的调调"），Agent 生成初稿，用户逐段反馈修改，Agent 迭代到满意。

  覆盖的知识点：
  - 风格控制：Few-shot / In-context learning 控制写作风格、语调、视角
  - 结构化生成：大纲 → 章节 → 段落的多层级生成，保持故事一致性（人物设定、时间线、伏笔）
  - 上下文一致性：前文说角色左撇子，后文不能变右撇子
  - 用户反馈理解："这里不够紧张"是抽象反馈，Agent 要能转化为具体修改
  - 多版本对比：同一场景生成 2-3 个版本让用户选
  - 迭代优化循环：生成 → 用户反馈 → 定位问题 → 重写 → 验证一致性
  - 素材管理：角色卡、世界观设定、伏笔清单的独立维护
  - 多样性 vs 质量权衡：temperature 控制 vs 约束条件（不能为追求创意编造矛盾）

  8. 竞品情报 Agent（搜索 + 聚合 + 洞察）

  需求：持续追踪指定竞品的动态——产品更新、融资消息、招聘方向、用户评价、社交媒体声量，定期生成竞品情报简报。

  覆盖的知识点：
  - 持续监控（Cron + Agent）：定时触发，不是按需触发
  - 多源信息采集：产品页变更检测、应用商店评价、GitHub commit、招聘信息、社交媒体
  - 信号提取：从招聘JD推断产品方向、从用户差评推断竞品弱点、从融资消息推断战略重心
  - 趋势分析：声量变化曲线、功能迭代时间线、用户满意度趋势
  - 影响评估：竞品动作对我们的影响程度（"竞品降价 20%" vs "竞品发了个小更新"）
  - 竞争格局映射：自动生成功能对比矩阵、价格对比图
  - 预警机制：重大变动实时通知（竞品发布核心功能、核心人员离职、重大融资）
  - 报告自动生成：周报/月报的模板化生成，关键指标自动更新
